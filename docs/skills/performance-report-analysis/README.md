# Performance Report Analysis

This playbook helps interpret test results and turn them into useful technical and stakeholder-facing reports.

## Use It For

- analyzing k6, JMeter, Gatling, Locust, or Artillery output
- identifying likely bottlenecks
- classifying severity
- writing technical findings
- writing business-facing summaries

## Start Here

- Main workflow: [SKILL.md](./SKILL.md)
- Tool-specific output guidance: [references/TOOL-REPORT-FORMATS.md](./references/TOOL-REPORT-FORMATS.md)
- Bottleneck patterns: [references/BOTTLENECK-PATTERNS.md](./references/BOTTLENECK-PATTERNS.md)
- Report templates: [references/REPORT-TEMPLATES.md](./references/REPORT-TEMPLATES.md)

## Example Prompt

```text
Use docs/skills/performance-report-analysis as the workflow.

Analyze this k6 summary and produce a technical report plus a stakeholder summary.
```
